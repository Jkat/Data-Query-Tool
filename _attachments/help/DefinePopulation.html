Help: Define Filters

Filters can be applied on any field in order to limit the set of results returned by a query, e.g. to extract data collected only at one specific site, or from a particular visit.

The "Define Filters" tab is organized like "Define Fields", where data fields are grouped per instrument or category. Select a category, then click on a field of interest to add it to the list of Selected Filters at bottom.
Finally, in the Selected Filters panel, define the filter criteria on that field via the "Operator" and "Value" columns. More than one filter may be added to a query. Multiple filters will be applied using 'and' logic.
The operators are as follows:

  • = equal to
  • != does not equal
  • <= less than
  • >= greater than
  • startsWith: filter for values starting with a specific character or string
  • contains: filter for values containing a specific character or string

Alternatively, a specific set of candidates or candidate visits can be defined as a filter by uploading a CSV file containing a list of candidate IDs with optional visit label (via the "Choose File" button).

Example: to query data collected at visit V01, select "demographics" from the Choose Category dropdown, then scroll down and click on "demographics,Visit_label". This field should now be displayed in the Selected Filters panel below. Next, define the filter constraint under the "Operator" and "Values" columns, e.g. select "=" and enter "V01".

Once filters have been defined, continue to the "View Data" tab.

Notes on querying behavioral data in LORIS

To extract only fully administered and fully complete instrument datasets, include the following field filters for each instrument in your query:

  • Administration = "All" or "Partial" (Note that "Partial" Administration poses a potential caveat for analysis)
  • Data_entry = "Complete"
  • DDE_complete = "Y" (for instruments where Double Data Entry applies)
  • Conflicts_Exist = "N" (all data entry quality issues have been resolved)

Additionally, under 'demographics' category:

  • Current_stage = "Visit"
  • Failure = "Neither" (to exclude both Failed and Withdrawn candidates)
